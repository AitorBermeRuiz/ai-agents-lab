### **SECTION 2: Problem & Solution (Context)**

Provides context on the business need and the technical response.

#### **2.1: The Business Problem**

A secure, automated personal finance management system was needed without relying on third-party apps that require direct bank account access or costly subscriptions.

#### **2.2: The Technical Challenge**

To build a hybrid solution that was:
* **Secure:** No bank credentials shared.
* **Automated:** No repetitive manual data entry.
* **Intelligent:** Ready for conversational AI analysis (via Claude AI).
* **Scalable:** Architected for future feature growth.

#### **2.3: The Implemented Solution**

A hybrid system combining Google Sheets (as the DB), Google Apps Script (for automation), and a custom **C# .NET 8 MCP Server** acting as the secure bridge between the data and an AI model (Anthropic's Claude).

### **SECTION 3: Technical Expertise & Implementation**

This is the core "fodder" for the RAG. Each point is a specific, retrievable skill.

#### **3.1: Backend Development (C# / .NET 8 / AI Integration)**

* **Architecture:** Designed and built a complete **ASP.NET Core** application following **Clean Architecture** and **SOLID** principles (Controllers, Services, Clients).
* **Protocol Implementation:** Implemented the **Model Context Protocol (MCP)**, an emerging standard from Anthropic for AI model tool-use. This is a key skill for **AI Engineering**.
* **API Design:** Created secure **RESTful** endpoints to serve the MCP tools.
* **Core Skills:** Leveraged **Dependency Injection** (Microsoft.Extensions.Hosting), clear abstractions (Interfaces + Implementations), and robust **async/await** programming.
* **AI Tools:** Developed 4 functional MCP tools for the AI: `get_ranges`, `write_range`, `get_range_formulas`, and `write_formulas`, all with validation.

#### **3.2: Cloud Integration & Security (Google Cloud Platform)**

* **Authentication:** Implemented **OAuth 2.0** (server-to-server flow) from scratch using **Service Accounts** for secure Google Cloud API access.
* **API Consumption:** Integrated the **Google Sheets API v4** for all data manipulation.
* **Security:** Managed granular scopes, token lifecycles (including refresh tokens), and project credentials in the Google Cloud Console.
* **Error Handling:** Built robust error handling and logging for all external API calls.

#### **3.3: Automation & Scripting (Google Apps Script)**

* **Automation:** Wrote 550+ lines of optimized JavaScript (ES6+) to handle monthly data archiving and sheet cleanup.
* **DevOps:** Implemented time-based **triggers** (cron-like jobs) to run the archival process on the 1st of every month.
* **Logic:** Included complex data validation and handling for European number formats.

#### **3.4: Data Modeling & UI/UX**

* **Data Structure:** Designed a relational-like data model across 4 interconnected sheets (Expenses, Budget, Dashboard, History).
* **Business Logic:** Implemented complex **SUMIFS** formulas and `SPARKLINE` progress bars directly in the sheets for real-time dashboarding.
* **UI Design:** Created a professional, dark-mode dashboard with custom SVG icons and a clear navigation system, focusing on high readability and usability.
